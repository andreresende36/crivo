# MCP Environment Variables Inspector

> Audit, validate, compare, and generate `.env` files from source code usages.

## Installation

```bash
npm install
npm run build
```

## Configuration

Add to your `.claude/settings.json`:

```json
{
  "mcpServers": {
    "env-inspector": {
      "command": "node",
      "args": ["/absolute/path/to/mcp-05-env-inspector/dist/index.js"],
      "cwd": "/your/project/root"
    }
  }
}
```

## Tools

### `list_env_vars`

Reads all `.env*` files and returns variable names with **masked values** (first 3 chars + `***`):

```
list_env_vars()
list_env_vars(unmask: true)   # shows full values — use with caution
```

Reads: `.env`, `.env.local`, `.env.development`, `.env.production`, `.env.staging`, `.env.test`, `.env.example`

### `check_missing_vars`

Scans all source files for `process.env.VAR_NAME` and `import.meta.env.VAR_NAME` usages,  
then reports which variables are **not defined** in any `.env` file:

```
check_missing_vars()
check_missing_vars(rootDir: "apps/api")
```

Returns: missing variables with the files that use them, and defined variables with which `.env` files contain them.

### `compare_environments`

Compares variable keys across all `.env*` files to find mismatches:

```
compare_environments()
```

Returns a matrix showing which vars are `defined` or `missing` in each environment file.  
Useful for catching vars that exist in `.env.local` but are missing from `.env.production`.

### `validate_configuration`

Checks that required variables are defined. Sources the required list from:
1. Explicit `requiredVars` parameter
2. `REQUIRED_ENV` array in `CLAUDE.md`
3. `.env.example` keys (fallback)
4. All scanned `process.env.*` usages (last resort)

```
validate_configuration()
validate_configuration(requiredVars: ["DATABASE_URL", "JWT_SECRET", "REDIS_URL"])
```

Returns `PASS` or `FAIL` with a list of missing variables.

### `generate_env_example`

Scans all source files, collects every `process.env.*` and `import.meta.env.*` usage,  
and generates a `.env.example` file with empty values and inline comments:

```
generate_env_example()              # returns content only
generate_env_example(write: true)   # writes to .env.example
```

Generated file groups variables by prefix (e.g. `DATABASE_`, `NEXT_PUBLIC_`, `AUTH_`)  
and includes comments showing which source files use each variable.

```env
# Auto-generated by mcp-env-inspector
# Generated: 2025-01-15T10:30:00.000Z
# Fill in the values for your environment

# ── AUTH ──
# Used in: src/auth/middleware.ts, src/auth/service.ts
AUTH_SECRET=
# Used in: src/auth/google.ts
AUTH_GOOGLE_CLIENT_ID=

# ── DATABASE ──
# Used in: src/db/client.ts
DATABASE_URL=

# ── NEXT ──
# Used in: src/lib/api.ts +2 more
NEXT_PUBLIC_API_URL=
```
